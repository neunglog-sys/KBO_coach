## 방법 A — 웹 브라우저로 빠르게 개발 ⭐ (UI 작업에 최적)

PC 브라우저에서 보면서 고치는 방식. 저장하면 즉시 새로고침(핫리로드). 캐릭터·입모션·TTS 모두 동작(데스크톱 크롬은 웹 TTS 지원).

```powershell
cd "C:\Users\82102_asozp43\OneDrive\바탕 화면\android\frontend"
npm run dev
```
→ 브라우저에서 **http://127.0.0.1:5000**

- 로그인: 데모 계정 **`admin` / `admin1234`** 로 통과 가능
- API(`/auth`, `/chat` 등)는 기본적으로 **로컬 백엔드(127.0.0.1:8000)** 로 프록시됨
  → 챗봇/데이터까지 쓰려면 **방법 B**(백엔드)도 같이 실행
- 폰 빌드 없이 UI를 빠르게 다듬을 때 가장 효율적

> 💡 **백엔드 없이 공개 서버로 붙이기**: `frontend/.env.development.local` 파일을 만들고 아래 한 줄 추가하면, 로컬 백엔드 없이도 챗봇이 동작한다.
> ```
> VITE_API_BASE=https://kboai-5dea0.web.app
> ```

## 방법 B — 백엔드 로컬 실행 (챗봇·데이터까지)

백엔드 의존성은 conda `project2` 환경에 있다.

```powershell
conda activate project2
cd "C:\Users\82102_asozp43\OneDrive\바탕 화면\android"
uvicorn main:app --app-dir services/api --host 0.0.0.0 --port 8000
```

- 확인: 브라우저에서 **http://127.0.0.1:8000/docs** 가 열리면 정상
- 루트 `.env` 에 `MONGO_URI`, `DATABASE_URL`, `LLM_API_KEY`, `JWT_SECRET` 등이 있어야 정상 동작
- `--host 0.0.0.0` : 같은 WiFi의 폰에서도 접근 가능하게 함 (127.0.0.1만 쓰면 PC 전용)
- 종료: 터미널에서 `Ctrl + C`

> 폰에서 로컬 백엔드로 붙이려면 추가로:
> 1. `frontend/.env.android` 의 `VITE_API_BASE` 를 `http://<PC_IP>:8000` 으로
> 2. Windows 방화벽 8000 인바운드 허용 (관리자 PowerShell):
>    ```powershell
>    New-NetFirewallRule -DisplayName "KBO Backend 8000 (LAN)" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 8000 -RemoteAddress LocalSubnet -Profile Any
>    ```
> 3. 폰·PC 같은 WiFi
> (그냥 공개 백엔드 쓰면 이 과정 전부 불필요)

## 방법 C — 폰 실기기에 빌드·설치 (네이티브 기능 확인)

웹에서 안 되는 것(네이티브 TTS 소리 등)은 실기기에서 확인. 이 앱은 `.env.android` 의 **공개 백엔드**로 붙으므로 폰만 인터넷되면 됨(방화벽·같은WiFi 불필요).

```powershell
# 0) 폰 USB 연결 + 설정 > 개발자옵션 > USB 디버깅 ON, "허용" 탭
cd "C:\Users\82102_asozp43\OneDrive\바탕 화면\android\frontend"

# 1) 웹 빌드(android 모드) + 네이티브 동기화
npm run android:sync

# 2) APK 빌드
cd android
.\gradlew.bat assembleDebug --no-daemon

# 3) 폰에 설치
$adb = "$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe"
& $adb install -r app\build\outputs\apk\debug\app-debug.apk

# 4) (선택) 마이크 권한 미리 부여
& $adb shell pm grant com.neunglog.baseballcoach android.permission.RECORD_AUDIO
```

- 코드 수정 후 반영: `npm run android:sync` 다시 → APK 재빌드 → 재설치
- **폰 화면을 PC 모니터로 보기**: 새 터미널에서 `scrcpy` (마우스로 조작도 됨)
- 생성 APK 위치: `frontend/android/app/build/outputs/apk/debug/app-debug.apk`

## 캐릭터 전환 (3D ↔ Live2D)

`frontend/src/components/MainView.tsx` 상단 import와 JSX만 교체:

```tsx
// ▼ 3D 사용
import Character3D from "./Character3D";
// import CharacterLive2D from "./CharacterLive2D";
...
<Character3D isSpeaking={isSpeaking} className="character" />

// ▼ Live2D 사용
// import Character3D from "./Character3D";
import CharacterLive2D from "./CharacterLive2D";
...
<CharacterLive2D isSpeaking={isSpeaking} className="character" />
```
바꾼 뒤 방법 A(웹) 또는 방법 C(폰)로 다시 실행.

## 한눈 요약

| 하고 싶은 것 | 명령 |
|---|---|
| **UI 빠르게 개발** | `cd frontend` → `npm run dev` → http://127.0.0.1:5000 |
| **백엔드 로컬** | `conda activate project2` → `uvicorn main:app --app-dir services/api --host 0.0.0.0 --port 8000` |
| **폰 앱 빌드·설치** | `npm run android:sync` → `cd android` → `gradlew assembleDebug` → `adb install -r ...` |
| **타입 체크** | `cd frontend` → `npm run typecheck` |
| **폰 화면 미러링** | `scrcpy` |

가장 흔한 흐름: **A로 UI 개발(+필요시 B) → 가끔 C로 폰 최종 확인.**

## 자주 막히는 곳

| 증상 | 원인 / 해결 |
|---|---|
| 웹에서 챗봇/로그인 실패 | 로컬 백엔드 미실행 → 방법 B 실행, 또는 `.env.development.local` 로 공개 백엔드 사용 |
| 폰 앱에서 데이터 안 옴 | `.env.android` 주소 확인, 폰 인터넷 확인 |
| `adb`에 기기 안 보임 | USB 디버깅 허용 팝업 확인 / 케이블 재연결 / `adb kill-server && adb start-server` |
| `device unauthorized` | 폰 잠금 해제 후 "허용" 탭 (scrcpy 번들 adb 충돌 시 SDK adb로 재시작) |
| APK 빌드 시 경로(한글) 에러 | `android/gradle.properties` 의 `android.overridePathCheck=true` 확인 |
| TTS 소리/입모션 없음 | 폰: 네이티브 TTS(이미 적용됨)·한국어 TTS 엔진 설치 확인 / 웹: 데스크톱 크롬에서 확인 |
